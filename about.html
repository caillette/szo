What is Szó

Szó means word in Hungarian. It displays questions and answers interactively, and comes with a vocabulary list to learn Hungarian from French, but you can use your own.

Szó just needs a decent Web browser to run. Vocabulary files are text files and you don't need an Internet connexion to use Szó, if you chose to install it locally. You can use the online version, too.

Installation

To install Szó, download the latest version and unzip it somewhere and open the index.html file in a Web browser (see "Supported configurations" below).

Usage

To chose a langage for the user interface, add the following to the URL: &lang=[code] (or ?lang=[code] if this is the first parameter of the URL). You can bookmark the URL to skip this step the next time. Available codes are:

  • hu: Hungarian (default),
  • en: English,
  • fr: French.

Vocabulary contains Cards which are a list of questions and answers. Cards have Tags which describe some kind of theme of usage for the Card. There can be several Tags for one Card.

On the left part of the display, there is a list of Tags to chose from.

By default, all the chosen Cards (corresponding to chosen Tags) show as a single list. To get interactive questions, uncheck List. Then click on Next to disclose answers or show another Card randomly taken from the list.

Define your own Vocabulary

Szó loads a Vocabulary file which references Pack files. Pack files are lists of Cards, which are some kind of plain-text sentences.

All those files are text files in UTF-8 encoding and Unix line breaks. To create or edit them, use a decent text editor like Notepad++ and take care of chosing UTF-8 encoding and Unix line breaks. Do not use Window's Notepad.

For Vocabulary and Pack file, any line becomes commented (its content being ignored) if starting by the hash (#) sign.

Pack and Vocabulary file names must end by .txt

Pack file

A Pack file contains Card definitions. A Card is a list of questions and answers keep together by indentations and no empty line. First question and first answer are unindented; other questions and answers are indented. There is at least one question and one answer per Card. A list of Tags (on the same line) may immediately precede the first question. The number of answer may be different than number of questions (there can be implicit questions or answers). Each Card has an optional list of Tags.

A Tag starts with an arrobas (@) and may contain any numerical or alphabetical character, without diacritics. It may also contain dash (-) or underscore (_). There must be no empty line between tags, questions, and answers of the same Card.

There can be a list of Tags applying to all the Cards in the Pack. This Tag list must appear first, with an empty line to separate it from Cards' tags.

Here is a sample of Pack file: 

@tag0 @tag1

  @tag2 @tag3
Question 1
  question 1.1
Answer 1
  answer 1.1
  answer 1.2

Question 2
Answer 2

In this sample we have the following Cards:

  • Card1 tagged with tag0 tag1 tag2 tag3
  • Card2 tagged with tag0 tag1 (global tags)

Vocabulary file

A Vocabulary file references all Pack files. In addition it may define some nice appellations for Tags (which are restricted to a very poor set of characters). The appellations will appear in place of the raw Tag name in the Tag selection area. Here is a sample referencing two Pack files, and two Tag appellations: 

fr-hu/file1.txt
fr-hu/file2.txt

@tag1 This is Tag one
@tag2 This is Tag two

Specifying alternate vocabulary file

It may be useful to define several Vocabulary files and switch between them, using a URL parameter: ?v=[relative file name]. The path must be relative to index.html file. Use &v= if this is not the first parameter in the URL. Of course such URL is bookmarkable.

Using with other languages

Szó may be useful for learning other languages.

As it reads UTF-8 vocabulary files, Szó accomodates with most of character sets, but it doesn't support right-to-left writing. If you need to add some characters to the supported set (mainly designed for French and Hungarian), please contact the author, Laurent Caillette.

Supported configurations

  • Safari 6.03 on Mac OS X 10.6.8: works fine.
  • Firefox 20.0 on Mac OS X 10.6.8: works fine.
  • Chrome 26 on Mac OS X 10.6.8: doesn't work with local files.

Other configurations should probably work, including on Windows and Linux. There will be no effort to make Szó run on any version of Internet Explorer.

The Chrome problem

(In simple words) Chrome forbids access to the local filesystem by default. Yes there is the allow-file-access-from-files command line option, that opens access to your filesystem to every other Web page, but since this options is potentially harmful Szó refuses to run with Chrome and file://.
